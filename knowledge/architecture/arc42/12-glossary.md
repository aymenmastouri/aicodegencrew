# 12 - Glossary

## 12.1 Business Terms

| Term | Definition |
|------|------------|
| ActionEntity | Core domain entity representing an action performed within the deed‑entry process. |
| ActionStreamEntity | Aggregates a sequence of actions for audit and replay purposes. |
| ChangeEntity | Captures a change request or amendment to an existing deed record. |
| ConnectionEntity | Models a logical connection between participants or documents. |
| CorrectionNoteEntity | Stores notes related to corrections applied to deed entries. |
| DeedCreatorHandoverInfoEntity | Holds hand‑over information generated by the deed creator subsystem. |
| DeedEntryEntity | Central entity representing a single deed entry in the registry. |
| DeedEntryLockEntity | Represents a lock used to prevent concurrent modifications of a deed entry. |
| DeedEntryLogEntity | Audit log entry for actions performed on a deed entry. |
| DeedRegistryLockEntity | Global lock entity ensuring consistency across the deed registry. |
| DocumentMetaDataEntity | Metadata describing documents attached to a deed (type, size, checksum). |
| FinalHandoverDataSetEntity | Finalised data set prepared for hand‑over to external authorities. |
| HandoverDataSetEntity | Intermediate data set used during the hand‑over workflow. |
| HandoverDmdWorkEntity | Work item representing a demand for hand‑over processing. |
| HandoverHistoryDeedEntity | Historical record of deeds that have been handed over. |
| HandoverHistoryEntity | General history of hand‑over operations. |
| IssuingCopyNoteEntity | Notes attached to copies issued from the original deed. |
| ParticipantEntity | Represents a participant (owner, buyer, authority) in a deed transaction. |
| RegistrationEntity | Entity that records the official registration of a deed. |
| RemarkEntity | Free‑form remarks associated with a deed or process step. |
| SignatureInfoEntity | Stores information about signatures applied to a deed (signer, timestamp). |
| SuccessorBatchEntity | Batch of successor deeds generated during succession processing. |
| SuccessorDeedSelectionEntity | Selection criteria for choosing successor deeds. |
| SuccessorDeedSelectionMetaEntity | Metadata describing the selection of successor deeds. |
| SuccessorDetailsEntity | Detailed information about a successor deed. |
| SuccessorSelectionTextEntity | Textual description used in successor selection UI. |
| UzvNumberGapManagerEntity | Manages gaps in the sequential UVZ number series. |
| UzvNumberManagerEntity | Central manager for UVZ number allocation. |
| UzvNumberSkipManagerEntity | Handles skipped UVZ numbers due to cancellations. |
| JobEntity | Represents a background job or scheduled task in the system. |

## 12.4 Architecture Patterns

| Pattern | Definition | Where Used |
|---------|------------|------------|
| Layered Architecture | Organises the system into logical layers (presentation, application, domain, data‑access, infrastructure). | All containers; presentation (Angular), application (service layer), domain (entities), data‑access (repositories). |
| Model‑View‑Controller (MVC) | Separates concerns of data (model), UI (view) and user interaction (controller). | Front‑end Angular components and Spring MVC controllers. |
| Service Layer | Provides an API for business use‑cases, orchestrating domain objects. | Application layer services (e.g., DeedEntryService). |
| Repository Pattern | Abstracts persistence operations behind a collection‑like interface. | Data‑access layer repositories (e.g., DeedEntryRepository). |
| Domain‑Driven Design (DDD) | Focuses on a rich domain model and bounded contexts. | Domain layer entities and aggregates (e.g., DeedEntryEntity, ParticipantEntity). |
| CQRS (Command Query Responsibility Segregation) | Separates read‑model queries from write commands. | Implemented via separate command services and query REST endpoints. |
| Event‑Driven Architecture | Uses events to decouple components and enable asynchronous processing. | Background jobs (JobEntity) and integration with external systems via messaging. |
| RESTful API | Exposes resources via standard HTTP verbs and stateless interactions. | All public REST interfaces (controllers). |
| Scheduler Pattern | Executes recurring tasks at defined intervals. | Scheduler component managing JobEntity executions. |
| Guard/Interceptor Pattern | Provides cross‑cutting concerns such as security and logging. | Route guards in Angular and Spring interceptors for API security. |
